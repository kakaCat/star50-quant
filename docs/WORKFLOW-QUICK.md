## 阶段1: 模型调参（只看IC）✅

```bash
python scripts/tune_xgb_model.py \
    --method bayesian --n_iter 50 \
    --ic_weight 1.0 --ir_weight 0.0 --return_weight 0.0 --drawdown_weight 0.0
```

**结果**: IC = 0.0953 (目标0.04) ✅

**为什么只看IC**: 
- IC衡量预测能力
- 交叉验证只能准确评估IC
- IR/收益/回撤需要在完整回测中评估

## 阶段2: 模型训练 ✅

用最佳参数训练最终模型，保存模型文件。

## 阶段3: 完整回测 ⏳

**关键阶段** - 这里才能得到真实的IR/收益/回撤

```bash
python scripts/run_backtest.py \
    --model_path tuning_results/.../xgb_best_model_*.json \
    --start_date 2020-01-02 \
    --end_date 2024-12-31
```

**评估指标**:
- IC: 预测能力（应保持~0.09）
- IR: 风险调整收益（目标>=1.5）
- 年化收益: 策略盈利（目标>35%）
- 最大回撤: 风险控制（目标<=-20%）

**决策**:
- ✅ 全部达标 → 跳到阶段5
- ⏳ 部分达标 → 阶段4优化
- ❌ 全部未达标 → 回阶段1

## ⚠️ 关键理解

### 为什么阶段1没达到IR/收益/回撤目标？

**不是因为调参方法有问题**，而是：

1. **阶段1只应该看IC** ✅
   - 交叉验证只能准确评估预测能力
   - 用简化策略评估收益是错误的

2. **IR/收益/回撤要在阶段3评估** ⏳
   - 需要完整的时序回测
   - 真实的交易规则和成本
   - 连续的时间序列

3. **策略优化在阶段4** ⏳
   - 通过调整策略参数提升收益
   - 通过组合优化提升IR
   - 通过风险控制降低回撤

### 类比

```
阶段1 = 训练射手的瞄准能力
阶段3 = 真实比赛中测试表现
阶段4 = 优化比赛策略

不能用训练场的模拟器（简化策略）
来评估真实比赛的成绩（IR/收益/回撤）！
```
