## 📚 三种方法详细说明

### 1️⃣ 随机搜索 (Random Search)

#### 原理
- **完全随机**地从参数空间中采样
- 每次试验的参数选择**互相独立**
- 不利用之前试验的结果

#### 示例过程

**参数空间**:
```python
{
    'max_depth': (3, 10),      # 3到10之间
    'learning_rate': (0.005, 0.3),
    'subsample': (0.5, 1.0),
    ...
}
```

**执行过程**:
```
试验1: max_depth=5,  learning_rate=0.15, subsample=0.7  → 评分: 0.45
试验2: max_depth=8,  learning_rate=0.03, subsample=0.9  → 评分: 0.52
试验3: max_depth=4,  learning_rate=0.22, subsample=0.6  → 评分: 0.38
试验4: max_depth=7,  learning_rate=0.08, subsample=0.85 → 评分: 0.58
...
```

**特点**:
- ✅ 简单直接，易于理解
- ✅ 适合快速探索参数空间
- ❌ 效率较低，需要较多试验次数
- ❌ 不利用历史信息

**使用命令**:
```bash
python scripts/tune_xgb_model.py --method random --n_iter 100
```

---

### 2️⃣ 网格搜索 (Grid Search)

#### 原理
- **遍历所有**参数组合
- 完整、系统、可重现
- 适合小参数空间

#### 示例过程

**参数网格** (简化版):
```python
{
    'max_depth': [4, 6, 8],           # 3个值
    'learning_rate': [0.01, 0.05, 0.1],  # 3个值
    'subsample': [0.7, 0.8, 0.9],     # 3个值
    'reg_alpha': [0, 1, 2]            # 3个值
}
```

**总组合数**: 3 × 3 × 3 × 3 = **81个组合**

**执行过程**:
```
试验1:  max_depth=4, lr=0.01, subsample=0.7, reg_alpha=0  → 0.42
试验2:  max_depth=4, lr=0.01, subsample=0.7, reg_alpha=1  → 0.45
试验3:  max_depth=4, lr=0.01, subsample=0.7, reg_alpha=2  → 0.48
...
试验81: max_depth=8, lr=0.1,  subsample=0.9, reg_alpha=2  → 0.55
```

**特点**:
- ✅ 完整遍历，不遗漏
- ✅ 结果可重现
- ❌ **组合爆炸**：我们的10个参数 → 数千个组合
- ❌ 效率极低，时间成本高

**为什么没用网格搜索？**

我们的参数空间：
```python
{
    'max_depth': (3, 10),          # 假设取5个值 = 5
    'learning_rate': (0.005, 0.3), # 假设取5个值 = 5
    'subsample': (0.5, 1.0),       # 假设取5个值 = 5
    'colsample_bytree': (0.5, 1.0),
    'colsample_bylevel': (0.5, 1.0),
    'min_child_weight': (1, 15),
    'gamma': (0.0, 1.0),
    'reg_alpha': (0.0, 3.0),
    'reg_lambda': (0.0, 5.0),
    'num_boost_round': (50, 300)
}
```

**总组合数**: 5^10 = **9,765,625** 个组合！

即使每个组合只需10秒，也要：
- 9,765,625 × 10秒 = 97,656,250秒
- = 1,627,604分钟
- = **3.1年**！

**使用命令**:
```bash
python scripts/tune_xgb_model.py --method grid
# 实际上我设计的代码会警告组合数过多
```

### 3️⃣ 贝叶斯优化 (Bayesian Optimization) ⭐

#### 原理
- **智能搜索**：利用之前所有试验的结果
- 使用**概率模型**预测哪些参数组合可能更好
- 平衡**探索**（尝试新区域）和**利用**（在好区域深挖）

#### 技术细节

使用 **TPE (Tree-structured Parzen Estimator)** 算法：

1. **建立概率模型**：根据历史试验，估计"好参数"和"坏参数"的分布
2. **选择下一个点**：选择最有可能改善结果的参数组合
3. **更新模型**：用新试验结果更新概率模型
4. **重复**：继续迭代

#### 示例过程

```
初始阶段（随机探索10次）:
试验1:  max_depth=5,  lr=0.15 → 评分: 0.45
试验2:  max_depth=8,  lr=0.03 → 评分: 0.52
试验3:  max_depth=4,  lr=0.22 → 评分: 0.38
...
试验10: max_depth=6,  lr=0.08 → 评分: 0.58

【TPE分析】发现：lr在0.05-0.1时表现较好，max_depth在5-7时较好

智能搜索阶段:
试验11: max_depth=6,  lr=0.07 → 评分: 0.61  ✓ 更好！
试验12: max_depth=7,  lr=0.06 → 评分: 0.63  ✓ 继续改善！
试验13: max_depth=5,  lr=0.09 → 评分: 0.59

【TPE分析】继续优化...

试验20: max_depth=7,  lr=0.088 → 评分: 0.65  ✓ 最佳！
```

**特点**:
- ✅ **最高效**：50次试验就能找到好参数
- ✅ **智能**：会重点搜索有希望的区域
- ✅ **平衡探索与利用**
- ✅ **适合高维空间**（我们有10个参数）
- ❌ 算法复杂，需要额外依赖(optuna)

**使用命令**:
```bash
python scripts/tune_xgb_model.py --method bayesian --n_iter 50
```

## 📊 三种方法对比总结

| 特性 | 随机搜索 | 网格搜索 | 贝叶斯优化 ⭐ |
|------|----------|----------|--------------|
| **效率** | 中等 | 极低 | **最高** |
| **需要试验数** | 100-200 | 数千-数万 | **50-100** |
| **适用场景** | 初步探索 | 小参数空间 | **复杂高维空间** |
| **是否智能** | 否 | 否 | **是** |
| **是否利用历史** | 否 | 否 | **是** |
| **结果可重现** | 否 | 是 | 部分是 |
| **实现难度** | 简单 | 简单 | 中等 |

### 我的选择

**主要用贝叶斯优化**，理由：
1. ✅ 10个参数 → 高维空间 → 贝叶斯最适合
2. ✅ 50次试验就能找到好参数 → 高效
3. ✅ 实际耗时15分钟 → 可接受
4. ✅ 找到的参数质量好 → IC=0.0953

**网格搜索为什么不用？**
- ❌ 组合数过多（数百万个）
- ❌ 需要3年才能跑完
- ❌ 完全不现实

**随机搜索为什么少用？**
- ⚠️ 需要更多试验（100-200次）才能达到贝叶斯50次的效果
- ⚠️ 时间成本2-4倍

## 🎯 如何选择调参方法？

### 决策流程

```
参数数量？
│
├─ ≤3个参数 → 网格搜索（完整遍历）
│
├─ 4-6个参数 → 
│   ├─ 每个参数取值≤5 → 网格搜索
│   └─ 否则 → 随机搜索或贝叶斯
│
└─ ≥7个参数 → 贝叶斯优化 ⭐（我们的情况）
```

### 实际建议

**阶段1: 粗调（快速探索）**
```bash
python scripts/tune_xgb_model.py --method random --n_iter 50
```
- 快速了解参数空间
- 识别有效区域

**阶段2: 精调（智能搜索）**
```bash
python scripts/tune_xgb_model.py --method bayesian --n_iter 100
```
- 基于粗调结果
- 深入优化

**阶段3: 验证（局部网格）**
如果想验证某几个关键参数，可以缩小范围后用网格：
```python
# 只调2-3个最重要的参数
param_space = {
    'max_depth': [5, 6, 7],
    'learning_rate': [0.05, 0.08, 0.1]
    # 其他参数固定
}
```
