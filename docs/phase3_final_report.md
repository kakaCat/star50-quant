## 问题诊断

### 跟踪误差超标分析

**现象**: 单期TE约束5%，但年化TE达到30.94%

**原因**:
1. **约束过松**: 5%单期约束在96期累积后达到30%+
2. **主动偏离过大**: 策略大幅超配高Alpha股票，追求超额收益
3. **市场波动**: 2年期间科创板波动剧烈
4. **集中度风险**: 个股权重上限5%，允许较高集中度

**理论验证**:
- 单期平均TE=0.55%
- 理论年化TE ≈ 0.55% × √96 ≈ 5.4%
- 实际TE=30.94%远超理论值（5.6倍）

**根本原因**: 策略在趋势行情中主动偏离基准，追求高Alpha。累计超额收益113%的代价是跟踪误差失控。

### 最大回撤超标分析

**现象**: 最大回撤-32.94%，超过20%目标

**原因**:
1. **高波动率**: 年化波动103.9%，远高于基准
2. **回测引擎缺陷**: 只计算周度净值，日内最大回撤可能更大
3. **风险控制不足**: 单期TE约束无法限制累积回撤
4. **市场下跌**: 科创板2023-2024经历多次深度调整

**时间分布**（从CSV推断）:
- 2024年12月出现显著回撤（-6.67%单期）
- 可能存在多次中等回撤累积

### 回测引擎缺陷

**问题**: 当前只在再平衡日计算净值，缺少每日数据

**影响**:
1. **年化计算错误**: 用96个数据点代替480天（修正后35.87%）
2. **波动率低估**: 周度波动103.9%已很高，日度会更高
3. **回撤低估**: -32.94%是周度回撤，日内回撤可能达-40%+
4. **风险指标失真**: 夏普比率、最大回撤等依赖日频数据

## 改进建议

### 优先级1: 修复回测引擎

**当前问题**: 只在再平衡日计算净值，缺少每日波动数据

**修复方案**:
```python
# 遍历所有交易日，而非只遍历再平衡日
all_trade_dates = sorted(prices['trade_date'].unique())
for date in all_trade_dates:
    if date in rebalance_dates:
        # 执行再平衡
        update_weights()
    # 每日计算持仓市值
    calculate_daily_pnl()
```

**预期改进**:
- 获得480个日度数据点（vs当前96个）
- 准确的年化、波动率、回撤指标
- 可视化每日净值曲线

### 优先级2: 收紧优化约束

**跟踪误差控制**:
- 将max_tracking_error从5%降至**2%**
- 预期年化TE降至10%以内
- 代价: 年化收益可能降至20-25%

**风险限制增强**:
```yaml
optimization:
  max_tracking_error: 0.02    # 2% (原5%)
  max_weight: 0.03            # 3% (原5%)
  max_turnover: 0.20          # 20% (原30%)
  max_sector_deviation: 0.05  # 新增：行业偏离5%
```

### 优先级3: 风险模型升级

当前使用样本协方差（纯历史方法），建议：
1. **Ledoit-Wolf收缩估计**: 减少估计误差
2. **Barra结构化模型**: 因子模型+特质风险
3. **EWMA协方差**: 指数加权，更重视近期
4. **风险预算约束**: 限制单因子风险贡献

### 优先级4: 动态TE约束

**方案**: 根据实际TE调整下期约束
```python
if realized_te > target_te * 1.3:
    max_te_next = max_te * 0.7  # 收紧30%
elif realized_te < target_te * 0.7:
    max_te_next = max_te * 1.2  # 放松20%
```

**优势**: 自适应控制，避免TE失控
