## 🔍 三种方法详细对比

### 方法1: 随机搜索 (Random Search)

**原理**: 在参数空间中随机采样

**优点**:
- ✅ 简单直接
- ✅ 易于实现
- ✅ 适合快速探索

**缺点**:
- ❌ 不利用历史信息
- ❌ 需要更多试验次数（100-200次）
- ❌ 效率较低

**适用场景**:
- 初步探索参数空间
- 建立性能基线
- 参数空间不太复杂

**示例**:
```bash
python scripts/tune_xgb_model.py \
    --method random \
    --n_iter 100 \
    --start_date 2020-01-02 \
    --end_date 2024-12-31
```

---

### 方法2: 网格搜索 (Grid Search)

**原理**: 遍历所有参数组合

**优点**:
- ✅ 完整遍历
- ✅ 不会遗漏
- ✅ 结果可重现

**缺点**:
- ❌ **组合爆炸**: 10个参数 → 数百万组合
- ❌ 时间成本极高（需要数年）
- ❌ 完全不适合高维空间

**我们的情况**:
```
参数数量: 10个
假设每个参数取5个值: 5^10 = 9,765,625 组合
假设每个组合10秒: 9,765,625 × 10秒 = 3.1年！
```

**结论**: ❌ **不可行**，只适合小参数空间（≤3个参数）

**示例**（仅用于演示，实际不建议运行）:
```bash
python scripts/tune_xgb_model.py --method grid
# 会警告组合数过多
```

---

### 方法3: 贝叶斯优化 (Bayesian Optimization) ⭐

**原理**: 使用概率模型智能搜索

**技术**: TPE (Tree-structured Parzen Estimator)

**过程**:
1. 前N次（10次）随机探索
2. 建立概率模型
3. 预测哪些参数可能更好
4. 选择最有希望的点尝试
5. 更新模型，继续迭代

**优点**:
- ✅ **最高效**: 50次就能找到好参数
- ✅ **智能**: 利用所有历史信息
- ✅ **适合高维**: 我们的10参数空间
- ✅ **平衡探索与利用**

**缺点**:
- ⚠️ 算法复杂
- ⚠️ 需要额外依赖(optuna)

**示例**（实际使用的命令）:
```bash
python scripts/tune_xgb_model.py \
    --method bayesian \
    --n_iter 50 \
    --start_date 2020-01-02 \
    --end_date 2024-12-31 \
    --cv_folds 5
```

**实际效果**:
- 耗时: ~15分钟
- 结果: IC=0.0953（优秀）
