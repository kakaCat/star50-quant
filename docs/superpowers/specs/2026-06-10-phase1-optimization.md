# Phase 1 优化方案

## 问题诊断

### 当前问题
- **IC日度均值**: -0.0099 (目标: >0.025)
- **IC>0比例**: 45.68% (低于50%)
- **特征数量**: 124个（可能过多）

### 问题分析
1. **时序不稳定**: 整体IC=0.0505但日度均值为负，说明存在严重的时间不一致性
2. **可能的未来信息泄露**: 截面特征(zscore/rank)可能使用了同期全部数据
3. **特征过多**: 124个特征可能包含噪音和冗余
4. **模型过拟合**: 训练集RMSE持续下降但验证效果差

## 优化策略

### 1. 特征重要性分析

**目标**: 识别真正有预测能力的特征

**方法**:
- LightGBM特征重要性排序
- 计算每个特征的单因子IC
- 分析特征间相关性
- 识别冗余特征

**输出**:
- Top 50特征列表
- 特征重要性报告
- 相关性矩阵

### 2. 特征选择与过滤

**目标**: 从124个特征筛选到50-80个核心特征

**筛选标准**:
- 特征重要性 > 阈值
- 单因子IC绝对值 > 0.01
- 与已选特征相关性 < 0.95
- IC时序稳定性（正向天数比例 > 50%）

**分类保留**:
- 交叉特征: 保留15-20个
- 时序特征: 保留10-15个
- 非线性特征: 保留5-10个
- 截面特征: 保留5-10个
- Alpha因子: 保留10个

### 3. 修复时序泄露问题

**问题定位**:
```python
# 当前代码（可能有问题）
result[col_name] = result.groupby('factor_date')[factor].rank(pct=True)
```

**问题**: `groupby('factor_date')`会在同一天内对所有股票排序，这在实际预测时是正确的。但如果训练时使用了未来日期的分布信息，就会泄露。

**修复方案**:
- 确保截面特征只使用当天数据
- 添加时序验证：检查特征是否依赖未来数据
- 使用滚动窗口代替全局统计

### 4. 改进特征计算逻辑

**优化点**:

1. **时序特征**: 增加滞后期，避免当期数据
```python
# 当前
group['momentum_5'] = group['close'].pct_change(5)
# 优化
group['momentum_5'] = group['close'].pct_change(5).shift(1)  # 使用昨天的动量
```

2. **截面特征**: 使用历史分布
```python
# 当前: 使用当天全市场分布
result[col_name] = result.groupby('factor_date')[factor].rank(pct=True)

# 优化: 使用历史20天分布的移动平均
rolling_rank = result.groupby('ts_code')[factor].rolling(20).rank(pct=True)
```

3. **Alpha因子**: 检查是否有未来信息

### 5. 模型正则化

**目标**: 防止过拟合

**方法**:
- 增加LightGBM正则化参数
- 减少树的深度
- 增加最小叶子样本数
- 使用更强的特征采样

**参数调整**:
```python
params = {
    'learning_rate': 0.05,  # 降低学习率
    'max_depth': 5,  # 限制深度
    'min_child_samples': 100,  # 增加最小样本
    'subsample': 0.8,  # 特征采样
    'colsample_bytree': 0.8,  # 列采样
    'reg_alpha': 0.1,  # L1正则
    'reg_lambda': 1.0,  # L2正则
}
```
